---
title: "Apply iTerm2 Color Schemes in Vim and Neovim editors"
description: "Install any of the 485+ auto-generated Vim colorscheme files from the vim/ directory and apply them with a single :colorscheme command."
---

Every color scheme in the collection has a corresponding Vim colorscheme file in the `vim/` directory. These files are auto-generated from the source `.itermcolors` data and define highlight groups for syntax, the UI, diff markers, spell checking, and more—so the color scheme looks consistent whether you're in a terminal or a GUI-capable Vim build.

## Installing a colorscheme

<Steps>
  <Step title="Locate the vim/ directory">
    The `vim/` directory at the root of the repository contains one `.vim` file per scheme (e.g., `vim/Dracula.vim`, `vim/Nord.vim`). File names use hyphens to replace spaces, matching the scheme's display name.
  </Step>
  <Step title="Copy the file to ~/.vim/colors/">
    Vim looks for colorschemes in `~/.vim/colors/` (or `~/.config/nvim/colors/` for Neovim). Copy the file you want:

    <CodeGroup>

    ```bash Vim
    cp vim/Dracula.vim ~/.vim/colors/
    ```

    ```bash Neovim
    cp vim/Dracula.vim ~/.config/nvim/colors/
    ```

    </CodeGroup>
  </Step>
  <Step title="Apply the colorscheme">
    In a running Vim or Neovim session, apply the scheme with the `:colorscheme` command, using the scheme name without the `.vim` extension:

    ```vim
    :colorscheme Dracula
    ```
  </Step>
  <Step title="Make it permanent (optional)">
    Add the `colorscheme` line to your `vimrc` (or `init.vim` / `init.lua`) so it loads on startup:

    <CodeGroup>

    ```vim ~/.vimrc or ~/.config/nvim/init.vim
    colorscheme Dracula
    ```

    ```lua ~/.config/nvim/init.lua
    vim.cmd("colorscheme Dracula")
    ```

    </CodeGroup>
  </Step>
</Steps>

## Colorscheme file structure

Each generated `.vim` file follows the same layout. Here is the complete `vim/Dracula.vim` as a representative example:

```vim vim/Dracula.vim
" Vim Color File
" Generated by iTerm2-Color-Schemes
" Name: Dracula
" URL: https://github.com/mbadolato/iTerm2-Color-Schemes

hi clear
if exists("syntax_on")
  syntax reset
endif

let g:colors_name = "Dracula"

" Normal colors
hi Normal guifg=#f8f8f2 guibg=#282a36

" Syntax highlighting groups
hi Comment guifg=#6272a4
hi Constant guifg=#f1fa8c
hi Identifier guifg=#bd93f9
hi Statement guifg=#ff6e6e
hi PreProc guifg=#ff92df
hi Type guifg=#69ff94
hi Special guifg=#a4ffff
hi Underlined guifg=#d6acff gui=underline
hi Error guifg=#ff5555 guibg=#282a36
hi Todo guifg=#ffffa5 guibg=#282a36

" Additional groups for better compatibility
hi String guifg=#50fa7b
hi Function guifg=#bd93f9
hi Keyword guifg=#ff6e6e

" Cursor and line highlighting
hi Cursor guifg=#282a36 guibg=#f8f8f2
hi CursorLine guibg=#44475a

" Visual selection
hi Visual guibg=#44475a

" Search highlighting
hi Search guibg=#ffffa5 guifg=#282a36

" Line numbers
hi LineNr guifg=#6272a4
hi CursorLineNr guifg=#f8f8f2

" Status line
hi StatusLine guifg=#f8f8f2 guibg=#6272a4

" Diff highlighting
hi DiffAdd guibg=#50fa7b
hi DiffChange guibg=#f1fa8c
hi DiffDelete guibg=#ff5555
hi DiffText guibg=#bd93f9
```

<Note>
  All colors in the generated files are in GUI hex format (`guifg`, `guibg`). They render correctly in terminal Vim when `termguicolors` is enabled (`set termguicolors`), and natively in GUI Vim builds such as MacVim or gVim.
</Note>

## Available schemes

The `vim/` directory contains 485 files, one for each scheme in the collection. File names correspond directly to scheme names with spaces replaced by hyphens—for example, `Tomorrow Night Eighties` becomes `vim/Tomorrow-Night-Eighties.vim`.
